# ClawMind 首次启动配置向导

## 功能说明

首次启动配置向导为用户提供友好的初始化体验，无需手动编辑配置文件。

## 向导流程

### 步骤 1: 欢迎页
- 展示 ClawMind 的核心功能
- 说明配置所需时间（约 2 分钟）

### 步骤 2: API 配置
- **选择提供商**：
  - OpenAI (GPT-4, GPT-3.5)
  - Anthropic (Claude 3)
  - 自定义（兼容 OpenAI API）

- **输入 API Key**：
  - 密码输入框保护隐私
  - 显示获取链接提示

- **选择模型**：
  - 根据提供商自动推荐
  - 支持自定义输入

- **自定义端点**（仅自定义提供商）：
  - API 端点 URL
  - 认证头配置

### 步骤 3: 服务启动
- 自动保存配置到 `%APPDATA%/ClawMind/config.json`
- 启动 Hermes Agent 服务
- 启动 OpenClaw Engine 服务
- 实时显示服务状态
- 错误提示和重试

### 步骤 4: 完成
- 显示配置摘要
- 确认服务运行状态
- 进入主应用界面

## 技术实现

### 前端 (Vue3)
- `SetupWizard.vue` - 向导组件
- 4 步进度指示器
- 表单验证
- 状态管理

### 后端 (Rust/Tauri)
- `save_config()` - 保存配置到文件系统
- `load_config()` - 加载现有配置
- `start_hermes()` - 启动 Hermes 进程
- `start_openclaw()` - 启动 OpenClaw 进程

### 配置存储
**Windows:**
```
C:\Users\<用户名>\AppData\Roaming\ClawMind\config.json
```

**Linux:**
```
~/.config/ClawMind/config.json
```

**macOS:**
```
~/Library/Application Support/ClawMind/config.json
```

## 配置文件格式

```json
{
  "provider": "openai",
  "apiKey": "sk-...",
  "model": "gpt-4",
  "apiEndpoint": "https://api.openai.com/v1",
  "authHeaderName": "Authorization",
  "authHeaderValuePrefix": "Bearer ",
  "websocketPort": 8765
}
```

## 用户体验

### 首次启动
1. 用户双击 ClawMind 图标
2. 检测到无配置文件
3. 自动显示配置向导
4. 用户完成 4 步配置
5. 服务自动启动
6. 进入主界面

### 后续启动
1. 用户双击图标
2. 检测到配置文件存在
3. 跳过向导，直接启动
4. 自动连接服务

### 重新配置
用户可以在主界面的设置中：
- 修改 API Key
- 切换模型
- 更改提供商
- 重置配置（触发向导）

## 验证逻辑

### API Key 验证
- 非空检查
- 最小长度检查（10 字符）

### 模型名称验证
- 非空检查
- 格式检查（字母、数字、连字符）

### 端点验证
- URL 格式检查
- HTTPS 协议检查（生产环境）

## 错误处理

### 配置保存失败
- 显示错误消息
- 允许重试
- 提供手动配置路径

### 服务启动失败
- 显示详细错误信息
- 检查端口占用
- 检查二进制文件存在
- 提供诊断建议

### 网络连接失败
- 检测 WebSocket 连接
- 自动重试机制
- 显示连接状态

## 测试场景

### 正常流程
1. 首次启动 → 显示向导
2. 选择 OpenAI → 自动填充端点
3. 输入有效 API Key → 启用下一步
4. 启动服务 → 成功进入主界面

### 异常流程
1. API Key 为空 → 禁用下一步
2. 服务启动失败 → 显示错误
3. 配置保存失败 → 允许重试
4. 用户中途关闭 → 下次重新显示

### 边界情况
1. 配置文件损坏 → 重新初始化
2. 端口被占用 → 提示更换端口
3. 磁盘空间不足 → 显示错误
4. 权限不足 → 提示管理员权限
